# LSP Phase 3 — TypeScript Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement task-by-task. Steps use checkbox syntax.

**Goal:** TypeScript/JavaScript code intelligence in the Structure editor via `typescript-language-server`, reusing all of P1+P2's frontend features. Adds the two things rust-analyzer didn't need: a **runtime** (Node) detected from the user's toolchain, and an **npm-based install** (the server + `typescript` are npm packages, not a single binary).

**Architecture:** Extends P1+P2 (shipped, v0.8.138). The registry gains an optional npm install method + runtime requirement alongside the existing single-binary path (rust-analyzer stays untouched). `crates/lsp` gains `runtime.rs` (login-shell Node/npm detection — the macOS GUI-PATH gotcha the P1 spec flagged). Everything downstream (client, manager, cm6, settings, diagnostics/completion/rename/actions) already works per-language from P2 — P3 is mostly backend + a few frontend registry/language-map lines.

**Tech Stack:** `typescript-language-server` + `typescript` (npm), the user's `node`/`npm` (≥18), tar not needed (npm does the unpack).

**Spec:** `docs/superpowers/specs/2026-07-07-lsp-integration-design.md` (P3 row: npm-registry tarballs, run via user's node, Node ≥18).

- vitest from repo ROOT; `cargo test -p karl-lsp`; app crate is `covenant`.

```rust
use std::path::PathBuf;
use std::process::Command;

use crate::LspError;

#[derive(Debug, Clone)]
pub struct RuntimeReq {
    pub name: String,        // "node"
    pub min_version: String, // "18"
    pub version_arg: String, // "--version"
}

#[derive(Debug, Clone)]
pub struct Resolved {
    pub path: PathBuf,
    pub version: String,
}

/// Resolve a runtime binary on the USER's login-shell PATH. GUI apps on
/// macOS inherit a minimal PATH, so we must ask the login shell (`-lc`)
/// rather than trust our own env — same class of problem the PTY env has.
pub fn detect(req: &RuntimeReq) -> Result<Resolved, LspError> {
    let shell = std::env::var("SHELL").unwrap_or_else(|_| "/bin/sh".into());
    // `command -v` prints the resolved path; then run --version.
    let out = Command::new(&shell)
        .args(["-lc", &format!("command -v {}", req.name)])
        .output()
        .map_err(|e| LspError::Spawn(format!("login shell: {e}")))?;
    let path_str = String::from_utf8_lossy(&out.stdout).trim().to_string();
    if !out.status.success() || path_str.is_empty() {
        return Err(LspError::RuntimeMissing {
            name: req.name.clone(),
            min: req.min_version.clone(),
            found: None,
        });
    }
    let path = PathBuf::from(&path_str);
    let ver_out = Command::new(&path)
        .arg(&req.version_arg)
        .output()
        .map_err(|e| LspError::Spawn(format!("{} {}: {e}", req.name, req.version_arg)))?;
    let version = String::from_utf8_lossy(&ver_out.stdout).trim().to_string();
    if !version_ge(&version, &req.min_version) {
        return Err(LspError::RuntimeMissing {
            name: req.name.clone(),
            min: req.min_version.clone(),
            found: Some(version),
        });
    }
    Ok(Resolved { path, version })
}

/// True if `found` (e.g. "v18.19.0") is >= `min` (e.g. "18" or "18.2")
/// on a major[.minor] comparison. Tolerates a leading `v`.
pub fn version_ge(found: &str, min: &str) -> bool {
    fn parts(s: &str) -> Option<(u32, u32)> {
        let s = s.trim().trim_start_matches('v');
        let mut it = s.split('.');
        let major = it.next()?.parse::<u32>().ok()?;
        let minor = it.next().and_then(|m| m.parse::<u32>().ok()).unwrap_or(0);
        Some((major, minor))
    }
    let (Some((fmaj, fmin)), Some((mmaj, mmin))) = (parts(found), parts(min)) else {
        return false;
    };
    (fmaj, fmin) >= (mmaj, mmin)
}
```
Add the error variant to `lib.rs`:
```rust
#[error("runtime {name} not found or too old (need >= {min}, found {found:?})")]
RuntimeMissing { name: String, min: String, found: Option<String> },
```
