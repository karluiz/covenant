# ACP Tab (A2) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** An opt-in `kind:"acp"` tab hosting an interactive long-lived Copilot ACP session rendered as a structured agent stream (prose, tool cards, interactive permission cards), wired end-to-end like the existing pi chat tab.

**Architecture:** Extend `AcpSession` with a Defer/respond permission seam (Task 1); add `acp_commands.rs` mirroring `pi_commands.rs` — registry on AppState, forwarder task emitting `session://{id}/acp`, `acp_event_to_phase` → `NotchHub::set_phase` (Task 2); typed api.ts wrappers (Task 3); `AcpChatView` mirroring `PiChatView` with the spike mockup's visual language (Task 4); tab-manager wiring + ⌘⌥⇧C shortcut (Task 5).

**Tech Stack:** Rust/tokio/serde (backend), TypeScript/vitest (frontend). **Zero new dependencies.**

**Spec:** `docs/superpowers/specs/2026-07-02-acp-tab-design.md` — decisions there bind this plan (hybrid permissions, pi-parity entry point, minimal v1 surface, pi-style persistence, aomExcluded).

      let choice = karl_agent::acp::policy::resolve_headless(req);

  - `PermissionPending {..}` → `Waiting { reason: "permission".into() }`
