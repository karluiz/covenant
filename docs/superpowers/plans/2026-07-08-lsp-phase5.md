# LSP Phase 5 — Java (Eclipse JDT.LS) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox syntax.

**Goal:** Java code intelligence in the Structure editor via the Eclipse JDT Language Server (jdtls), reusing P1–P4's frontend. This is the most launch-complex server: a **tar.gz** download, a **Java 21+** runtime, and a launch that spawns `java` with a big JVM-flag set + the equinox `-jar` launcher + a **writable copy of the arm64 config dir** + a **required `-data` workspace dir**. No project-load handshake (unlike C#) — `rootUri` + `-data` auto-imports; readiness is signaled by a `language/status {ServiceReady}` notification.

**Architecture:** Extends P1–P4 (shipped, v0.8.140) additively. Distribution facts are in `.superpowers/lsp-p5-research.md` (verified live — actually downloaded + spawned jdtls + got an initialize response + ServiceReady). **Task 3's smoke is the hard gate**: real tarball download + java spawn + ServiceReady + a resolved Java definition, or the phase BLOCKS.

**Tech Stack:** Eclipse JDT.LS `1.60.0` (tarball), the user's **Java 21+** (JDT.LS 1.60 requires `JavaSE 21`; Java 17 will NOT launch it — verified).

## Verified distribution facts (from research — do not re-derive)

- Download (pinnable milestone): `https://download.eclipse.org/jdtls/milestones/1.60.0/jdt-language-server-1.60.0-202606262232.tar.gz`, 50,925,681 bytes, sha256 `e94c303d8198f977930803582738771fd18c52c5492878410bf222b1aa81ef1d` (matches published `.sha256`).
- Launcher jar: `plugins/org.eclipse.equinox.launcher_1.7.200.v20260619-2039.jar` (pin the exact path — the version is fixed by the JDT.LS pin).
- Config dir (arm64 mac): **`config_mac_arm`** (NOT `config_mac`, which is Intel). Both exist in the tarball; arm64 must use `config_mac_arm`.
- **Runtime: Java 21+ REQUIRED** — JDT.LS 1.60 OSGi bundles need `JavaSE;version=21`; Java 17 dies silently (no initialize response, `BundleException: Unresolved requirement version=21` in the eclipse log). `java --version` → stdout `openjdk 17.0.18 …` (version is the 2nd token; 1st is "openjdk"). `java -version` → stderr `openjdk version "17.0.18" …` (quoted).
- **`-configuration` MUST be a WRITABLE copy** of `config_mac_arm` (jdtls extracts a JNI helper into it; read-only → `AccessDeniedException`, crash). **`-data <workspace dir>` is REQUIRED** (omitting → `IllegalStateException`, exit 13). Both must be per-server writable dirs.
- Working launch (verified, initialize in ~2.2s, ServiceReady ~2.25s): `java -Declipse.application=org.eclipse.jdt.ls.core.id1 -Dosgi.bundles.defaultStartLevel=4 -Declipse.product=org.eclipse.jdt.ls.core.product -Dfile.encoding=UTF-8 -Xmx1G --add-modules=ALL-SYSTEM --add-opens java.base/java.util=ALL-UNNAMED --add-opens java.base/java.lang=ALL-UNNAMED -jar <launcher.jar> -configuration <writable config copy> -data <workspace dir>`. stdio works with no extra flag.
- Readiness: `language/status` notification, terminal `{"type":"ServiceReady"}`. No explicit project-load step — `rootUri`/`workspaceFolders` + `-data` triggers auto-import (pom.xml confirmed). Root markers: `pom.xml` (confirmed), `build.gradle`, `settings.gradle`, `.project` (inferred).

Keep `version_ge`'s comparison. Java uses `version_arg: "--version"` (stdout). This must not regress node/dotnet (their first version-ish token is still found first).

### Task 2: Java launch branch (writable config copy + data dir + JVM args)

**Files:** `crates/app/src/lsp_commands.rs`.

**Interfaces:** A FOURTH spawn shape for java (jdtls). `lsp_start` for a java server (`install_kind()==Binary && spec.runtime.name=="java"` && `spec.config_subpath.is_some()`): detect java (≥21, presence+version gate — needs-runtime already generic from P4), then per-server:
1. Copy `<install_root>/<config_subpath>` → `<data_dir>/lsp/jdtls/<id>/config/` (WRITABLE — jdtls writes there). Recursive dir copy.
2. Create `<data_dir>/lsp/jdtls/<id>/data/` (the `-data` workspace dir).
3. Spawn `java.path` (the resolved runtime) with args: `[<JVM flags...>, "-jar", <install_root>/<entry_subpath>, "-configuration", <config copy dir>, "-data", <data dir>]` where JVM flags = the verified set (`-Declipse.application=org.eclipse.jdt.ls.core.id1 -Dosgi.bundles.defaultStartLevel=4 -Declipse.product=org.eclipse.jdt.ls.core.product -Dfile.encoding=UTF-8 -Xmx1G --add-modules=ALL-SYSTEM --add-opens java.base/java.util=ALL-UNNAMED --add-opens java.base/java.lang=ALL-UNNAMED`). cwd = the workspace root.

The existing 3-way spawn (npm→node, C#→apphost+roslyn-args, plain-binary→rust) gains this 4th arm. Distinguish java by `spec.config_subpath.is_some()` (only jdtls sets it) or `spec.runtime.name=="java"`. rust/TS/C# arms unchanged.
