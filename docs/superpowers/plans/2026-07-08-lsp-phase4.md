# LSP Phase 4 — C# (Roslyn) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox syntax.

**Goal:** C# code intelligence in the Structure editor via the Roslyn-based `Microsoft.CodeAnalysis.LanguageServer`, reusing P1–P3's frontend. New-vs-prior: a **zip (.nupkg) download with a nested entry**, a **runtime precondition (.NET 10) that gates a directly-spawned apphost** (not launched *through* a runtime), **required launch args** (`--logLevel`, `--extensionLogDirectory`, `--stdio`), and a **C#-specific `solution/open` handshake** after `initialize` (without it the server loads no project and features stay empty).

**Architecture:** Extends P1–P3 (shipped, v0.8.139) additively. Verified distribution facts are in `.superpowers/lsp-p4-research.md` (read it — everything was curl/unzip/handshake-verified live). The one UNVERIFIED piece is the exact `solution/open` params — **Task 3's smoke is the empirical gate: it discovers the working handshake and proves a real C# definition resolves, or the phase BLOCKS.**

**Tech Stack:** Roslyn LSP `Microsoft.CodeAnalysis.LanguageServer.osx-arm64@5.4.0-2.26179.14` (NuGet .nupkg), the user's `.NET 10` runtime.
