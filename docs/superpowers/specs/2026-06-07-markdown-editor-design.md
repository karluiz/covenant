# MarkdownEditor ÔÇö lightweight WYSIWYG for Covenant textareas

**Date:** 2026-06-07
**Status:** Approved (design)
**Author:** Karluiz + Claude

## Problem

Several markdown-bearing surfaces use raw `<textarea>`s that feel crude for how
central they are:

- **Operator editor Ôćĺ "The Soul"** (`op-soul-body`): big textarea editing the
  SOUL.md **body** (the YAML front-matter is produced separately by the
  Identity/Behaviour structured controls). A right-hand pane redundantly shows a
  `marked` preview plus the full raw SOUL.md.
- **Project Notes Ôćĺ docs** (`docs-tab.ts`): textarea + `marked` preview behind an
  edit/preview icon toggle.
- **Hard constraints** (`op-soul-hard`): small textarea, short rule lists.
- **Spec Creator composer**: a single-line-growing chat input.

We want a **real WYSIWYG** (no visible markdown symbols) that is **lightweight**,
vanilla (no React/heavyweight framework ÔÇö per CLAUDE.md pitfall #9), theme-aware
(must go neutral black under True Dark), and reusable across all four sites.

## Decision summary

| Question | Decision |
|---|---|
| Editor engine | **Milkdown** (ProseMirror-based) ÔÇö true WYSIWYG |
| Formatting UX | **Markdown input rules only** ÔÇö no toolbar, no bubble, no slash menu; ÔîśB/ÔîśI keymap |
| Scope | All four call sites |
| Operator right pane | **Remove the rendered preview**; keep a small `[ editor ┬Ě .md ]` source toggle; keep the collapsible full SOUL.md source (it shows generated YAML ÔÇö distinct info) |

## Architecture

### Component: `ui/src/ui/markdown-editor.ts`

A vanilla class wrapping Milkdown, mirroring `CustomSelect` ergonomics
(`ui/src/ui/select.ts`):

```ts
interface MarkdownEditorOptions {
  value: string;
  placeholder?: string;
  mode?: "full" | "inline";      // default "full"
  onChange?: (md: string) => void;
  onSubmit?: () => void;          // inline only: Enter fires this
}

class MarkdownEditor {
  readonly element: HTMLElement;  // mount this
  get value(): string;
  set value(md: string);
  focus(): void;
  destroy(): void;
}
```

- **Lazy load:** Milkdown core + preset-commonmark + ProseMirror deps are pulled
  via dynamic `import()` (same pattern as `marked` today in `docs-tab.ts` and
  `operators.ts`), so the initial bundle is unaffected ÔÇö weight is paid only when
  an editor is first opened. Construction is therefore async-internally: the
  `element` is returned synchronously (empty shell) and the editor crank mounts
  into it once the dynamic import resolves; `value` set before ready is buffered.
- **`full` mode:** commonmark preset ÔÇö headings, lists, blockquote, inline code,
  bold/italic/links. Input rules: `## ` Ôćĺ heading, `- ` / `* ` Ôćĺ bullet list,
  `1. ` Ôćĺ ordered list, `> ` Ôćĺ blockquote, `` `x` `` Ôćĺ code, `**x**`/`*x*` Ôćĺ
  bold/italic. Keymap: ÔîśB, ÔîśI.
- **`inline` mode:** marks only (bold/italic/code) + plain paragraphs, no block
  transforms. Enter calls `onSubmit`; Shift+Enter inserts a newline. Used by the
  Spec composer and Hard constraints.

### Theming

Milkdown's bundled theme is **not** used. We ship `ui/src/ui/markdown-editor.css`
(or a block in an existing sheet) styling the ProseMirror surface with Covenant
tokens: `--bg-overlay`, `--text`, `--border`, `--accent`, `--muted`,
`rgba(var(--ink-rgb), ÔÇŽ)` for overlays. This makes True Dark and Light work with
no extra branching.

### Markdown round-trip

Serialization uses Milkdown's remark serializer. Operator edits touch **only the
body**, so the YAML front-matter is never round-tripped through ProseMirror ÔÇö it
stays byte-exact as generated by the structured controls. Body prose may be
normalized (e.g. `*` bullets Ôćĺ `-`, collapsed blank-line runs); this is
acceptable for freeform prose and documented here as expected behavior.

## Call-site integration

### 1. Operator "The Soul" (`ui/src/settings/operators.ts`)

- Replace the `op-soul-body` textarea with a `MarkdownEditor` (`full`) bound to
  `view.body`; `onChange` Ôćĺ `view.body = md; commit(false)`.
- Add a small header toggle `[ ÔŚž editor ┬Ě <> .md ]` (reuse the icon-toggle visual
  language introduced for docs-tab) that swaps the WYSIWYG view for a raw
  body-markdown `<textarea>` and back, keeping content in sync.
- **Remove** the right-pane rendered `marked` preview (`op-soul-preview` +
  `renderPreview()`), now redundant.
- **Keep** the collapsible "SOUL.md source" (`op-soul-rawwrap` / `op-soul-source`)
  ÔÇö it shows the full generated file including YAML front-matter, which the
  WYSIWYG does not.

### 2. Project Notes docs (`ui/src/project-notes/docs-tab.ts`)

- Replace the textarea + `marked` preview + edit/preview icon toggle with a
  single `MarkdownEditor` (`full`). Remove the dual-view machinery
  (`setMode`, `renderPreview`, the two mode buttons). Persistence
  (`scheduleSave`/`flush`) stays, fed by `onChange`.

### 3. Hard constraints (`ui/src/settings/operators.ts`)

- Replace `op-soul-hard` textarea with `MarkdownEditor` (`inline`, compact
  height), bound to `view.hard_constraints`.

### 4. Spec composer (`ui/src/spec-chat/immersive.ts`)

- Replace the composer `<textarea>` with `MarkdownEditor` (`inline`).
  `onSubmit` Ôćĺ existing `submit()`. Preserve Enter=send / Shift+Enter=newline via
  the inline keymap. Auto-grow is handled by ProseMirror's natural height; cap
  with max-height CSS as today.

## Testing

- **`ui/src/ui/markdown-editor.test.ts`** (new): value get/set round-trip,
  `onChange` fires on edit, `onSubmit` fires on Enter in inline mode (not in
  full mode), `destroy()` removes listeners/DOM. Milkdown is heavy under jsdom ÔÇö
  mock or lazily skip the live editor where the crank can't mount, asserting the
  wrapper contract (buffering, async-ready) directly.
- Update existing tests that assume a textarea:
  `ui/src/settings/operators.test.ts`, `ui/src/project-notes/*.test.ts`,
  `ui/src/spec-chat/*` where they query the composer textarea.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| ProseMirror bundle weight | Dynamic `import()`; paid only on first open |
| Markdown normalization on round-trip | Body is freeform prose; documented; YAML never round-tripped |
| Enter semantics in composer | Explicit inline keymap (EnterÔćĺsubmit, Shift+EnterÔćĺnewline) |
| jsdom can't mount ProseMirror | Test the wrapper contract; mock the crank |
| z-index inside full-window modals | Editor mounts inline (not portaled), so unaffected; any future popovers must clear `2147482000` like the recent select fix |

## Out of scope

- Toolbars, bubble menus, slash commands (explicitly declined).
- Tables, images, embeds ÔÇö commonmark preset only for now.
- Collaborative editing.
