# 3.8 — Convergence Mode

## Goal

A "command room" overlay that renders every terminal session as a live tile so the user can see — at a glance, without tab-hopping — what each agent executor is doing, what state it's in, what the operator decided last for it, and **reply directly to any tile that escalated**, unblocking the session without leaving the overlay.

## Out of scope

- **Embedding inside AFK mode (3.4)** — Convergence stays standalone in v1. AFK absorption is a separate spec.
- **Editing agent prompts from a tile** — tiles are read-only **except for the reply box on `ESCALATED` tiles** (see 3.13).
- **Switching operators from a tile** — the operator picker (⌘⇧O) handles that. Tiles inherit the per-tab operator.
- **Tile drag / reorder / custom layout** — fixed CSS-grid auto-flow. Tile order = tab order.
- **Recording / replay / scrubbing** — separate spec.
- **Filtering / hiding tiles** — v1 shows all tabs. Per-tab "hide from Convergence" is v2.
- **Tile virtualization** — v1 caps usable density at ~12 tiles before scroll kicks in.
- **New per-tile shortcuts beyond reply** — only `⌘⇧M` (open/close), `Esc`, click-to-focus, and the reply box on escalated tiles.
- **Persistence/learning of replies** — the reply box only unblocks the session in 3.8. Persistence as memory is **3.13**.
- **Convergence Mode as a marketing surface** — tiles show real state always.

## Acceptance criteria

### Layout & navigation

- [ ] `⌘⇧M` toggles the Convergence Mode overlay (full-window, z-index above tab strip and sidebar — same layering as AFK). `⌘⇧O` remains the operator picker and is **not** changed by this spec.
- [ ] Layout: CSS grid `auto-fit, minmax(280px, 1fr)`, 16 px gap, 24 px outer padding. No JS layout calculation.
- [ ] Tiles update within 1.5 s of underlying state changes (1 s backend snapshot poll is acceptable for v1).
- [ ] Clicking a tile (outside the reply box) focuses that tab and exits the overlay.
- [ ] `Esc` and the explicit "Exit" button both close the overlay. If a reply box is focused, `Esc` blurs the box first; a second `Esc` closes the overlay.
- [ ] Empty state: zero tabs → centered "No sessions" message + Exit button. Shortcut still toggles.
- [ ] Overlay respects active vibrancy mode (uses `--bg-overlay`, no hardcoded colors).

### Tile content (top-to-bottom)

- [ ] **(1) Header**: tab title (truncated) + tab color stripe + **operator avatar** (the operator assigned to that tab, from 3.2 multi-operator state).
- [ ] **(2) Vendor badge**: detected AI CLI vendor for the tab — one of `claude`, `copilot`, `opencode`, `aider`, `codex`, or `unknown`. Detection is heuristic (parse the most recent foreground command). When `unknown`, render a neutral badge with the literal command name truncated. Vendor list lives in `crates/app/src/convergence.rs::detect_vendor()` with a unit test per supported vendor.
- [ ] **(3) Status pill**: one of `idle` / `working` / `awaiting-input` / `blocked` / `operator-thinking`.
- [ ] **(4) Last operator decision**: action label + one-line rationale, truncated to two lines.
- [ ] **(5) Last command + last non-empty output line preview** (ANSI-stripped, truncated).
- [ ] **(6) Reply box** — visible **only** when status is `blocked` (operator escalated). One-line text input + scope selector (`one-shot` default | `mission` | `global`) + "Send" button. Submitting:
      (a) sends the reply text to the escalated operator session (via Tauri command — implementation detail in plan), the session resumes;
      (b) closes the reply box on that tile;
      (c) the persistence side of (b) — turning the reply into a memory — is **owned by 3.13**, not this spec. 3.8 emits a structured `convergence_reply_submitted` event; 3.13 listens for it.
- [ ] **(7) Cost-this-session footer** ONLY when the tab is enrolled in AOM — `$0.42 / $5.00 budget`. Hidden otherwise.

### Backend / quality gates

- [ ] `npx tsc --noEmit` passes; `cargo check -p covenant` is clean.
- [ ] Rust unit tests cover the snapshot builder's status classifier AND the vendor detector (one test per supported vendor + one for `unknown`).
- [ ] No new color tokens. Status pill and vendor badge colors come from existing tokens (`--accent`, `--muted`, `--danger` if it exists).

## File boundaries

### Create

- `ui/src/convergence/overlay.ts` — `ConvergenceOverlay` class (mount/unmount/toggle, snapshot subscription, reply submit). ≤ 240 lines.
- `ui/src/convergence/tile.ts` — `renderTile` / `updateTile`, including reply box rendering for `blocked` tiles. ≤ 240 lines.
- `crates/app/src/convergence.rs` — `ConvergenceTileState`, `ConvergenceSnapshot`, `build_convergence_snapshot()`, `classify_status()`, `detect_vendor()`. Pulls from existing `operator.rs`, `aom.rs`, `session::Session` state. NO schema changes. ≤ 280 lines including tests.

### Touch

- `ui/src/main.ts` — `⌘⇧M` shortcut + Esc handling + instantiation. ≤ 30 lines added.
- `ui/src/styles.css` — overlay + tile + reply-box + vendor-badge styles, appended at end. ≤ 260 lines. Reuse existing tokens only.

### DO NOT touch

- `crates/agent/`, `crates/blocks/`, `crates/session/`, `crates/pty/`.
- `crates/app/src/operator.rs`, `aom.rs`, `safety.rs`, `settings.rs`, `storage.rs` — read-only consumers. The reply submit pipes through existing operator-session APIs; if a path doesn't exist, ESCALATE rather than widening these modules.
- `ui/src/operator/`, `ui/src/aom/`, `ui/src/recall/`, `ui/src/blocks/`, `ui/src/structure/`, `ui/src/tabs/`, `ui/src/settings/` — orthogonal surfaces. Convergence reads tab/operator metadata via small adapters, does not modify them.

## Complexity

`medium` — 3 AOM sessions. Largest risk areas: (a) the reply-submit pipe (must reach the right operator session without touching `operator.rs`'s public API), (b) vendor detection heuristic across edge cases (`npx`, aliases, wrappers).

## Resolved decisions

- **Update mechanism**: snapshot poll, 1 s tick from frontend. Single `get_convergence_snapshot` command.
- **Tile sort order**: tab position. "Activity sort" deferred to v2.
- **Cost footer when tab excluded from AOM**: hide row entirely, no chip.
- **Shortcut**: `⌘⇧M` (mission control). `⌘⇧O` stays as operator picker.
- **Multi-operator**: each tile inherits the operator already assigned to its tab. No operator switching from inside Convergence.
- **Reply box scope**: `one-shot` (default) | `mission:<id>` | `global`. `one-shot` does NOT persist as memory — only the latter two do, and the persistence is 3.13's job.
- **Vendor detection**: in scope. Heuristic only; `unknown` is a first-class state, never an error.
