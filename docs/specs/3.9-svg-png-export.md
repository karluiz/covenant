# 3.9 — SVG → PNG Quick Export

## Goal

Convert the open SVG file to a PNG sitting next to it with a single
click in the editor header — so the user can re-export to PNG as part
of their normal day-to-day flow without leaving Covenant or scripting
it themselves.

## Out of scope

- **Non-SVG image conversions** — no PNG→JPG, no PDF→PNG, no
  raster→raster. SVG source only.
- **Other export targets** — no PDF, no WebP, no AVIF. PNG only.
- **High-fidelity rendering via `resvg`** — Canvas API output is the
  v1 path. Adding `resvg` (or any Rust-side rasterizer) is a follow-up
  if Canvas fidelity proves insufficient.
- **Save-as dialog / clipboard / browser download** — destination is
  always `<basename>.png` next to the source SVG. Other destinations
  are deferred.
- **Background color picker** — fixed transparent background for v1.
  No "white background" toggle.
- **Custom dimensions / aspect override** — scale multipliers only
  (1x / 2x / 3x). No pixel-width input, no DPI control.
- **Batch export** — single open file only. No "export all SVGs in
  folder" workflow.
- **Export from the file tree without opening** — must open the file
  in the editor to trigger.
- **Markdown / image-tag rewrites** — exporting the PNG does NOT
  update any references in the user's docs.

## Acceptance criteria

- [ ] When a `.svg` file is open in the structure editor, the header
      shows a `[PNG]` button and a `[1x ▾]` scale dropdown next to the
      existing `source ↔ preview` toggle. Both controls are hidden for
      every other file type, including `.md`.
- [ ] Clicking `[PNG]` writes `<basename>.png` adjacent to the open
      SVG (overwriting if it already exists), using the in-memory SVG
      content (so unsaved edits are reflected in the export) and the
      currently selected scale multiplier.
- [ ] After a successful export, an info toast reads `Exported
      <basename>.png (<scale>x, <W>×<H>)` where W/H are the final
      pixel dimensions.
- [ ] If the SVG is malformed (parser error) or the write fails, an
      error toast reports the underlying message; no partial / empty
      `.png` is left on disk.
- [ ] Scale selection persists across app restarts via
      `localStorage` under `structure-png-scale` (default `2`).
- [ ] `⌘⇧P` continues to toggle source ↔ preview as today; the new
      controls do not steal that shortcut or any existing CM6 binding.
- [ ] Existing `.md` preview behaviour is unchanged (no PNG button
      appears for markdown).

## File boundaries

- **Create**: `ui/src/structure/png-export.ts` (≤ 80 lines) —
  pure-ish function `svgToPng(svgText: string, scale: 1|2|3):
  Promise<{ bytes: Uint8Array; width: number; height: number }>`.
  Renders via `Image` + `<canvas>` + `canvas.toBlob`. No DOM coupling
  beyond the transient canvas it creates.
- **Touch**: `ui/src/structure/editor.ts` (≤ 100 lines added) —
  add the two header controls, wire visibility to `previewKind ===
  "svg"`, handle click → call `svgToPng` → call new write API → toast.
- **Touch**: `ui/src/api.ts` (≤ 6 lines added) — wrapper
  `structureWriteBinaryFile(path: string, bytes: Uint8Array):
  Promise<void>` that invokes the new Rust command.
- **Touch**: `crates/app/src/structure.rs` (≤ 25 lines added) — new
  async fn `structure_write_binary_file(path: String, bytes: Vec<u8>)
  -> Result<(), String>` mirroring the existing text writer (same
  `spawn_blocking` discipline, same path validation).
- **Touch**: `crates/app/src/lib.rs` (1 line) — register the new
  command in the Tauri builder's `invoke_handler`.
- **Touch**: `ui/src/styles.css` (≤ 30 lines added) — visual style
  for `.structure-editor-png-btn` and `.structure-editor-png-scale`,
  matching the existing `.structure-editor-preview-btn` look.
- **DO NOT touch**: `ui/src/structure/preview.ts` — the SVG preview
  renderer is unrelated. PNG export reads the source string directly.
- **DO NOT touch**: any other file viewer / tab / sidebar code — the
  feature lives entirely inside the structure editor surface.

## Complexity

`small` (1 AOM session). One new pure module + one new backend command
+ two header controls. ~250 lines total across all files.

## Open questions

None. All design decisions resolved during brainstorm
(2026-05-03):
- Destination → adjacent `<basename>.png`, overwrite without confirm.
- Scale → 1x/2x/3x dropdown, persisted, default 2x.
- Background → transparent (alpha preserved).
- Rasterizer → browser Canvas (no `resvg` dep for v1).
- Feedback → toast on success and failure; no Finder reveal in v1.
