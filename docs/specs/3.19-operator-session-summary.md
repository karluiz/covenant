# 3.19 — Operator session summary (familiar chat command)

## Goal

Pedirle al familiar un resumen ejecutivo de la sesión actual —
especialmente útil tras una corrida larga del operador en AOM, donde
tomó decisiones autónomas por mí mientras dormía o estaba AFK. La
respuesta debe enfocarse en **qué decidió el operador, por qué, y
qué quedó bloqueado**, no en un dump cronológico de comandos.

## Tradeoff documentado (decisión 2026-05-05)

Se evaluaron dos opciones:

1. **Vista UI dedicada** (panel propio, atajo `⌘⇧S`): más
   descubrible, pero duplica superficie con los morning reports de
   AOM y exige diseño/estado nuevos.
2. **Comando dentro del chat del familiar** ✅ *elegida*: gratis sobre
   la infraestructura existente (`crates/familiar`), una sola
   superficie de conversación, sin UI nueva. Costo: menos
   descubrible — se mitiga con un hint en docs hub (3.5) y un
   placeholder en el input del chat ("prueba: /summary").

Si el uso sube y la falta de descubribilidad duele, promovemos a vista
dedicada en una spec posterior. **No anticipar.**

## Out of scope

- **Vista UI dedicada / atajo global** — explícitamente diferida.
- **Cambios al motor de summarization** (`summarizer.rs` Haiku/Sonnet
  ya cubre lo necesario). No tocar.
- **Cambios al schema de SQLite del familiar.** Se lee lo que ya hay
  (`events`, `summaries`, `missions`, `directives`, `costs`).
- **Reportes cross-session / cross-tab.** Esta spec es por sesión
  (tab) actual. Convergence (3.8) es la vista global.
- **Export a markdown / share.** Si surge, spec aparte.
- **Reemplazar morning reports de AOM.** Coexisten: AOM dispara
  reportes automáticos al terminar mission; `/summary` es
  on-demand desde chat.
- **Comandos slash genéricos del familiar.** Esta spec sólo agrega
  uno (`/summary`). El framework slash es scope futuro si crecen.

## Acceptance criteria

- [ ] En el chat del familiar, escribir `/summary` (o `/resumen`)
      produce una respuesta estructurada con: **decisiones autónomas
      tomadas**, **costos**, **bloqueos resueltos**, **misiones
      activas/cerradas**, **open items pendientes de humano**.
- [ ] El comando acepta scope opcional: `/summary` (sesión actual
      por defecto), `/summary mission` (mission activa),
      `/summary today` (ventana de 24h del familiar).
- [ ] La respuesta usa el rolling summary cacheado cuando existe;
      sólo invoca Sonnet lazy si el rolling está stale (>N eventos
      desde último rollup). N configurable, default 50.
- [ ] El comando respeta el cost cap diario del familiar; si está
      en frozen-mode, responde con el último summary cacheado más
      un disclaimer.
- [ ] El parser de comandos no rompe el flujo de chat normal: texto
      que no empieza con `/` sigue yendo al agente como mensaje.
- [ ] Tests: unit del parser de comando + integration con un
      `Llm` mock que verifique el prompt enviado incluye el scope
      correcto y excluye eventos fuera de ventana.
- [ ] Hint visible: placeholder del input del chat menciona
      `/summary` la primera vez que se abre el chat tras una
      sesión AOM completada (señal: hay rows en `missions` con
      `closed_at`).

## File boundaries

- **Touch**: `crates/familiar/src/agent.rs` (≤ 80 líneas) — añadir
  parser de slash command y branch al builder de prompt.
- **Touch**: `crates/familiar/src/prompts.rs` (≤ 60 líneas) —
  template del prompt de summary con secciones fijas.
- **Touch**: `crates/familiar/src/memory.rs` (≤ 40 líneas) — query
  helpers por scope (`session`, `mission`, `today`) si no existen
  ya. **No** cambiar schema.
- **Create**: `crates/familiar/tests/summary.rs` (≤ 200 líneas).
- **Touch**: `ui/src/familiar/chat.tsx` (o equivalente, ≤ 30 líneas)
  — placeholder condicional del input.
- **DO NOT touch**: `summarizer.rs`, `observer.rs`, `cost.rs`,
  `directive.rs`. Esta feature consume, no extiende, el motor.
- **DO NOT touch**: AOM morning report code. Ortogonal.

## Complexity

`small` (1 AOM session). Toda la plumería existe; es composición
sobre `crates/familiar`.

## Resolved decisions (2026-05-05)

- **Idioma de la respuesta**: sigue el idioma del último mensaje
  del chat.
- **Ventana de `/summary today`**: rodante de 24h (no día calendario).
- **Formato de salida**: markdown plano dentro del chat, sin bloques
  colapsables ni UI nueva.
