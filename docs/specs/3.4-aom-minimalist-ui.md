# 3.4 ‚Äî AOM Minimalist Mode ("Battery Mode")

> **DEPRECATED (2026-05-05).** AFK Mode is superseded by Convergence Mode (3.8), which
> covers the "glance at it while not actively driving" use case more densely
> (all sessions visible, with status, cost, mission snapshots, and inline reply
> on escalations). OS notifications (3.6) cover alerting when away from screen.
> AOM (the autonomous-overnight *engine*) remains; only this AFK *UI surface*
> is retired. The idle/screensaver role is absorbed by 3.8 ‚Äî see
> `3.8-convergence-mode.md` ("Idle/screensaver role" section) for auto-engage
> behavior. Existing 3.4 implementation (overlay, feed, banner) stays in the
> codebase for now but is no longer the canonical idle view; future work on
> idle UX lands in 3.8.

## Goal

Add an "AFK mode" UI for AOM sessions: a minimal full-window
overlay that surfaces only the operator's decision stream and
budget, designed for "leave it running overnight, glance at it in
the morning."

> Status note: the AOM engine is built (Phase B+C ‚Äî banner,
> reports, cost cap, `aom_sessions` storage, `‚åò‚áßR` ReportPanel).
> What's missing is the *minimalist live view* the vision asks for.
> The existing banner is for in-flight context; this spec adds a
> separate, takeover-style surface.

## Out of scope

- **Engine changes** ‚Äî pipeline, decision detection, safety, cost
  tracking are built. Touching them is out of scope; ESCALATE.
- **Cost / budget logic** ‚Äî already implemented; this spec only
  *renders* existing state.
- **Morning report panel** ‚Äî `‚åò‚áßR` already exists. AFK mode is
  *during* the run; the report is *after*.
- **OS notifications** ‚Äî see `3.6-os-notifications.md`. AFK mode
  may consume notification triggers but does not own them.
- **Multi-tab AFK** ‚Äî show all currently-AOM-enabled tabs in one
  feed; do NOT design a per-tab AFK overlay variant.
- **Auto-engage on idle** ‚Äî see open questions. v1 is manual
  toggle.
- **Persisting feed across restarts** ‚Äî feed is reconstructed from
  the existing `aom_sessions` rows on open; no new persistence.

## Acceptance criteria

- [ ] `‚åò‚áßA` toggles AFK mode (only enabled when at least one tab
      has AOM active).
- [ ] AFK is a full-window overlay (z-index above tab strip and
      sidebar) with three regions:
      (1) header ‚Äî total cost / budget / elapsed time / active tab
      count;
      (2) decisions feed ‚Äî chronological cards: timestamp, tab
      label, action (DECIDE / REPLY / ESCALATE), one-line
      rationale;
      (3) footer ‚Äî "Wake up" button + small "exit Esc" hint.
- [ ] Feed auto-scrolls to bottom on new decision; scrolling up
      pauses auto-scroll; "back to live" pill appears.
- [ ] `Esc` and "Wake up" both exit AFK to normal UI.
- [ ] AFK respects active vibrancy mode (uses `--bg-overlay`).
- [ ] Decisions in the feed link to the source block when clicked
      (focuses the right tab, scrolls block into view, exits AFK).
- [ ] When all AOM tabs finish/exit, AFK shows "Run complete ‚Äî
      open report?" prompt linking to `‚åò‚áßR`.
- [ ] `npx tsc --noEmit` passes; no new Rust code so
      `cargo check -p covenant` should not regress.

## File boundaries

- **Create**:
  - `ui/src/aom/afk.ts` ‚Äî `AfkOverlay` class, mount/unmount/feed
    update (‚â§ 320 lines).
- **Touch**:
  - `ui/src/main.ts` ‚Äî `‚åò‚áßA` shortcut + Esc handling. Max 30
    lines.
  - `ui/src/styles.css` ‚Äî overlay styles. Max 220 lines, appended
    at end. Reuse `--bg-overlay`, `--border`, `--muted`,
    `--accent`. No new color tokens.
  - `ui/src/aom/banner.ts` ‚Äî add link/button to enter AFK from
    the banner. Max 25 lines added.
  - `ui/src/api.ts` ‚Äî only if a new "subscribe to operator
    decisions stream" wrapper is needed AND the event already
    exists in Rust. ESCALATE if a new IPC event is required ‚Äî
    this spec assumes the existing event bus is sufficient.
- **DO NOT touch**:
  - `crates/` ‚Äî this is a frontend-only feature.
  - `ui/src/operator/`, `ui/src/recall/`, `ui/src/blocks/`,
    `ui/src/structure/`, `ui/src/tabs/`, `ui/src/settings/`.
  - `ui/src/aom/report.ts` ‚Äî leave the morning report panel
    untouched.

## Complexity

`small`-`medium` ‚Äî 1-2 AOM sessions. Largest risk: subscribing
to the live operator decisions stream from the UI without a new
IPC channel. If the existing stream is per-tab only, ESCALATE
before adding a multi-tab aggregator.

## Open questions

- **Auto-engage on idle**: enter AFK after N minutes of no
  keyboard input *and* at least one AOM tab active? (Probably
  yes, default 10 min, configurable. v2.)
- **Display dimming**: AFK mode could lower terminal brightness or
  request macOS prevent-display-sleep. v2.
- **Filter by tab**: should AFK feed allow filtering to a single
  tab? (v1: no, all-tabs.)
- **Empty state**: what shows when AFK is opened but no AOM is
  active? Currently the shortcut would be disabled; alternative
  is a "no AOM running ‚Äî start one?" prompt.
