# 3.8.1 — Convergence Redesign (Inbox + Roster)

## Goal

Replace the flat tab-grid Convergence overlay (3.8) with a two-pane **Inbox + Roster** layout that makes escalations the primary surface, collapses same-operator-many-tabs into a single roster row, hides tabs without an assigned operator, and gives the reply path a large multi-line composer with explicit affordances. Convergence becomes the place where the user replies to operators — not a passive dashboard.

## Why

The 3.8 grid has four field-validated UX failures:

1. **Escalated tiles are hard to find** — they share a single visual plane with idle/working tiles and only differ by a small status pill.
2. **Reply input is unusable** — small, embedded inside a noisy tile, overflows on long text, easy to miss.
3. **Click anywhere on the card focuses the tab** — there is no safe area to read or compose without leaving the overlay.
4. **Tile per tab regardless of assignment** — every open tab gets a tile, including tabs without an operator. Same operator across N tabs produces N redundant tiles.

The redesign treats Convergence as a **comms hub** (escalations to reply to + roster of who's running what), not a dashboard.

## Out of scope

- **Cross-operator context awareness** ("operator A reads what operator B is doing") — deferred to a separate v2 spec; only a placeholder note is left at the end of this doc.
- **Operator-to-executor chat threads** — current scope still routes replies to the *escalated* operator session via the existing 3.8 reply pipe. Free-form chat is a future spec.
- **Issue tracker integration (GitHub/internal)** — explicitly discarded for now.
- **AFK embedding (3.4)** — Convergence remains standalone.
- **Tile drag/reorder, recording/replay, virtualization** — same exclusions as 3.8.
- **Editing operator prompts from a roster row** — read-only; operator picker (`⌘⇧O`) handles assignment.
- **Persistence/learning of replies** — owned by 3.13. This spec only emits the same `convergence_reply_submitted` event.

## Acceptance criteria

### Layout & navigation

- [ ] `⌘⇧M` toggles the overlay (unchanged from 3.8). `Esc` closes; if a reply textarea is focused, first `Esc` blurs, second closes.
- [ ] Two-column layout: **Inbox (left, 1.3fr)** + **Roster (right, 1fr)**, single CSS grid, no JS layout calculation. On window width <960 px, columns stack (Inbox on top).
- [ ] Snapshot poll cadence unchanged (1 s). UI updates within 1.5 s of state change.
- [ ] Empty state: zero **assigned** operators → centered "No operators assigned" message + Exit button. (Note: assignment, not tab count, drives empty state.)
- [ ] Overlay respects active vibrancy mode (uses existing `--bg-overlay`, no new color tokens).

### Filtering & roster scope

- [ ] **Only tabs with an assigned operator appear anywhere in Convergence** (Inbox or Roster). Tabs without an assigned operator are excluded — no "unknown" tiles.
- [ ] **Roster groups by operator identity**: one row per distinct operator across all assigned tabs. A row showing N>1 sessions exposes a disclosure caret to expand inline sub-rows (one per tab) showing per-tab status. Default state: collapsed if all sub-rows are `idle`/`working`; auto-expanded if any sub-row is `escalated`.
- [ ] **Roster filter chips** (v1): `all` (default) | `escalated` | `working` | `idle`. Chips filter the roster only — Inbox always shows all escalations.
- [ ] Roster row sort order: (1) any escalated session inside the operator → top, (2) most recently active, (3) name alphabetical as final tiebreaker.

### Inbox (left column)

- [ ] Inbox lists one card per escalated **session** (not per operator) — if one operator has two escalated tabs, two Inbox cards.
- [ ] Inbox header: `Inbox · N awaiting you` (N hidden when 0; in that case render an empty-state message inside the column: "Nothing awaiting you").
- [ ] Each Inbox card shows: operator avatar + name, tab name, vendor badge (reused from 3.8), `ESCALATED` pill, time since escalation, and the operator's question (full text, wraps; no truncation in Inbox — truncation only in Roster).
- [ ] **One Inbox card is "active" at a time** (highlighted, reply composer expanded). The first card in list order is active by default. `↑`/`↓` move active card. Clicking a card makes it active.
- [ ] **Reply composer** (only on active card):
      - Multi-line `<textarea>`, auto-grow up to 8 lines then scroll, min 2 lines.
      - Scope selector: `one-shot` (default) | `mission` | `global` — same semantics as 3.8.
      - Submit: `⌘↵` or Send button. On submit: emit existing `convergence_reply_submitted` event (no new IPC), clear textarea, advance active card to next escalation (or fall back to "Nothing awaiting you" if list empties).
      - Escape blurs the textarea before closing the overlay.

### Roster (right column)

- [ ] Each row: avatar + operator name + summary status. If N>1 sessions: a `N sessions` chip and disclosure caret.
- [ ] Sub-row (expanded): tab name + per-tab status pill + last-command excerpt (truncated to one line). No reply box in roster sub-rows — replies happen in Inbox.
- [ ] **Click behavior — explicit and non-overlapping**:
      - **Single click** on a roster row (or sub-row) → focus that tab AND close the overlay.
      - **Double click** → focus that tab but **keep the overlay open**.
      - **Click on the disclosure caret only** → toggle expand/collapse, no focus change, no overlay close.
      - The Inbox column has *no* click-to-focus — clicking an Inbox card only sets it active for replying.
- [ ] Cost-this-session footer (when AOM-enrolled) appears inside the operator row, right-aligned. Hidden otherwise. Same rule as 3.8.

### Backend / quality gates

- [ ] `npx tsc --noEmit` passes; `cargo check -p covenant` is clean.
- [ ] Snapshot builder updated to:
      (a) drop tabs without an assigned operator,
      (b) group sessions by operator id and expose `OperatorRosterEntry { operator, sessions: Vec<SessionSummary> }`,
      (c) keep emitting an explicit `escalations: Vec<EscalationCard>` collection for the Inbox (flat list, sorted by escalated-at).
- [ ] Rust unit tests cover: assignment-filter (unassigned tabs excluded), grouping (same operator across 2 tabs → 1 entry with 2 sessions), escalation list ordering (oldest-first by escalated-at).
- [ ] No new color tokens. Escalation accent reuses existing danger color (introduce `--danger` if it does not exist — single token only).

## Complexity

`medium` — risk areas: (a) keyboard navigation across Inbox cards while a textarea is focused (textarea must intercept `↑`/`↓` for caret movement; nav only fires when textarea is empty or `⌘`-modified — pick one and document in plan), (b) the assignment-filter must not regress when a tab's operator is cleared mid-session (snapshot must drop it on next tick), (c) the disclosure-caret hit area must not bubble into the row click handler.

## Resolved decisions

- **Tile primary unit**: one row per assigned operator (operators with N>1 tabs collapse with disclosure).
- **Visibility filter**: tabs without an assigned operator are excluded entirely.
- **Layout**: Inbox (left) + Roster (right), 1.3fr / 1fr, stacked under 960 px.
- **Reply**: multi-line auto-grow textarea (min 2, max 8 lines), `⌘↵` submit.
- **Filter chips**: shipped in v1 (`all` / `escalated` / `working` / `idle`).
- **Click behavior**: single = focus + close, double = focus + keep open, caret = toggle only.
- **Inbox active card**: first by default, `↑`/`↓` to move, no auto-advance after submit aside from picking the next pending escalation.
- **Empty states**: separate copy for "no assigned operators" (overlay-level) and "nothing awaiting you" (Inbox-level).
