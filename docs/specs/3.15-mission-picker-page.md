# 3.15 — Mission Picker Full Page

## Goal

Promover el modal de "Set mission spec" a una página propia (`#mission-page`), siguiendo el patrón ya consolidado por Drafts (3.10) y Docs Hub (3.5).

El modal acumuló demasiada información — Published (14+), Superpowers (6+ con badges spec/plan), picker libre — y se siente apretado: las cards no respiran, no hay search, no hay preview del spec antes de confirmar, y la sección Superpowers compite por scroll con Published. Una página dedicada da espacio para search + preview pane y reduce la fricción de "abrir, leer, decidir".

## Out of scope

- Cambiar el contrato del Operator (sigue leyendo path → `.md` publicado).
- Editar specs desde la página — para escribir/editar se sigue usando Drafts (3.10).
- Persistir favoritos o orden aprendido (sort sigue siendo semántico por ID descendente).
- Multi-select / batch set para varias tabs.
- Sincronización con servicios externos.
- Eliminar el modal de un golpe — la página es el destino, el modal queda removido en la misma PR pero el atajo `⌘M` y el click en statusbar siguen siendo los entry points (sólo cambia el target).
- Reescribir el parser de specs publicados / superpowers (reusa lo existente de 3.11).

## Acceptance criteria

- [ ] Trigger del statusbar "Set mission" y atajo `⌘M` abren `#mission-page` como overlay full-screen (no modal), con animación `settings-fade` consistente con Drafts/Docs.
- [ ] Layout: header `56px` con título "Set mission" + close `×`; body en grid de 2 columnas — izquierda lista (sidebar ~360px), derecha preview pane que renderiza el `.md` seleccionado (markdown → HTML, mismo render que Docs Hub).
- [ ] Sidebar tiene search box sticky arriba (placeholder "Search specs…", filtra por ID + title + goal snippet, case-insensitive); debajo, secciones colapsables: **Published (N)**, **Superpowers (N)**, **Drafts (N)** (drafts mantiene el botón "Publish to use" del 3.11, no "Set"), y al final card "Or pick another file…" con text input + Browse.
- [ ] Click simple sobre una card la marca `selected` y carga el preview a la derecha sin confirmar. La card de la mission actual de la tab activa muestra badge `current` (mismo criterio path-exact-match de 3.11).
- [ ] Footer fijo abajo con `[Cancel] [Set mission]`; "Set mission" deshabilitado hasta que haya selección válida (card seleccionada o path no vacío en el input).
- [ ] Preview pane muestra el contenido del `.md` con scroll independiente; si el archivo no existe (path libre escrito a mano que aún no existe), muestra empty state "File not found — will be set as path-only mission". Si el archivo es > 200 KB, trunca con aviso.
- [ ] Atajos dentro de la página: `↑/↓` navegan cards de la sección activa, `Enter` confirma, `Esc` cierra y vuelve al estado previo, `⌘F` enfoca el search box, `Tab` mueve foco al input "or pick another file".
- [ ] Empty states: sin published muestra "No published specs yet. Write one in Drafts (⌘⇧D)" con link al panel; secciones sin items se ocultan en vez de mostrar "0".
- [ ] Carga: al abrir, `list_published_specs`, `list_superpowers_missions` y `list_drafts` se disparan en paralelo. Skeleton de 3 filas por sección mientras cargan; error inline + Retry si falla, con el text input + Browse siempre operativos como fallback.
- [ ] El modal anterior (`mission-picker.ts`) queda eliminado de la misma PR; cualquier callsite externo que lo invocaba pasa por el nuevo `openMissionPage(...)`.
- [ ] Tests TS cubren: filtrado del search (match por ID, title, goal), navegación con flechas entre secciones, `canSubmit` (último-gana entre card e input), ocultamiento de secciones vacías. No se duplican los tests Rust del 3.11.

- Backend: 0 líneas — todo el data layer ya existe (3.11).

## Open questions

- ¿La página se monta como overlay (z-index, no afecta tabs) o como "tab interna virtual" tipo Docs/Drafts? (Recomendación: overlay full-screen como Drafts ya lo es — consistencia visual y `Esc` para volver al estado previo sin tocar la tab activa.)
- ¿El preview renderiza markdown completo o sólo Goal + headings? (Recomendación: completo con scroll — el usuario quiere leer antes de decidir; si performance es un issue, lazy-render bajo el fold.)
- ¿Click en una card de Superpowers con badge `plan ✓` muestra el plan o el spec en el preview? (Recomendación: spec por default, toggle `Spec | Plan` arriba del preview cuando ambos existen.)
- ¿`⌘M` desde dentro de la página cierra (toggle) o no hace nada? (Recomendación: toggle — consistente con `⌘P` de Recall.)
