# 3.3 — Structure (optional file tree sidebar)

## Goal

Add an optional Zed-style file tree to the sidebar so the user can
peek at and edit a single file without leaving Covenant — the
"escape hatch" that prevents the user from having to open a real
IDE for a one-line change.

## Out of scope

- **LSP / diagnostics / autocomplete** — Covenant is not an IDE.
  An editor pane, if added, is a plain textarea or `<textarea>`-
  level surface. No tree-sitter, no language servers.
- **Multi-file tabs inside the editor pane** — one file at a time.
- **Refactor / rename / multi-file ops** — out.
- **Search across files** — Recall handles *command* history
  search; cross-file content search is its own spec, not this one.
- **Git status decorations on tree nodes** (modified, untracked) —
  v2.
- **File watcher with live updates** — v1 is manual refresh
  (button or re-expand). Adding a watcher means dealing with
  `node_modules`-class noise, which is a separate problem.
- **Folder manipulation** (create/delete/rename folders) — v1 is
  read-tree, edit-files only.
- **Following symlinks recursively** — first-level only; show as
  symlink badge but do not traverse.
- **Replacing the terminal as primary surface** — per the vision's
  "100% opcional" principle, the tree is hidden by default and the
  terminal is unaffected when it's open.

## Acceptance criteria

- [ ] Sidebar gains a third view "Structure", parallel to Blocks
      and Recall (the existing two sidebar views).
- [ ] Toggling Structure uses the existing sidebar nav pattern (no
      new global shortcut required).
- [ ] Tree root = active tab's `cwd`. Switching tabs re-roots the
      tree.
- [ ] Tree honors `.gitignore` patterns AND a hardcoded ignore set
      (`node_modules`, `.git`, `target`, `dist`, `.next`,
      `__pycache__`).
- [ ] Folders expand/collapse on click; expanded state persists
      per-cwd in `localStorage`.
- [ ] Clicking a file opens it in a minimal editor pane (right-of-
      sidebar, left-of-terminal, or as overlay — see open
      questions).
- [ ] Editor save (`⌘S`) writes the file via a Tauri command;
      success toast on save.
- [ ] Files larger than a configurable threshold (default 1 MB)
      show a "too large to preview" placeholder, NOT the contents.
- [ ] `npx tsc --noEmit` passes; `cargo check -p covenant` clean.

## File boundaries

- **Create**:
  - `ui/src/structure/tree.ts` — tree component (≤ 320 lines).
  - `ui/src/structure/editor.ts` — minimal editor (textarea-based)
    + save flow (≤ 220 lines).
  - `crates/app/src/structure.rs` — `list_dir`, `read_file_text`,
    `write_file_text` Tauri commands. `.gitignore` parsing via
    a minimal crate (`ignore` if already in deps; ESCALATE if not)
    or hand-rolled fallback (≤ 220 lines).
- **Touch**:
  - `ui/src/main.ts` — wire sidebar nav entry. Max 30 lines.
  - `ui/src/styles.css` — tree + editor styles. Max 200 lines,
    appended at end. Reuse `--bg-panel`, `--border`, `--muted`,
    `--accent`. **No new color tokens** unless ESCALATED.
  - `ui/src/api.ts` — wrap the new Tauri commands. Max 60 lines.
  - `crates/app/src/lib.rs` — register new commands. Max 15 lines.
  - `Cargo.toml` (workspace) — `ignore` crate ONLY if missing
    AND user confirms in Open Questions resolution.
- **DO NOT touch**:
  - `ui/src/blocks/`, `ui/src/recall/`, `ui/src/operator/`,
    `ui/src/aom/`, `ui/src/tabs/`, `ui/src/settings/`.
  - `crates/agent/`, `crates/blocks/`, `crates/session/`,
    `crates/pty/` — this is purely a frontend + thin filesystem
    backend.

## Complexity

`medium` — 2-3 AOM sessions. Largest risk: the editor pane.
Splitting it into "tree only (read-only)" and "editor pane" is
allowed; ship tree first.

## Open questions

- **Editor pane vs. shell-out**: should clicking a file open an
  in-app editor, or shell out to `$EDITOR` (or the `code` CLI)?
  In-app is more aligned with "don't leave Covenant"; shell-out
  is far simpler. ESCALATE before choosing.
- **Refresh strategy**: manual button vs. lightweight watcher
  scoped to expanded folders?
- **Performance ceiling**: at what tree size does v1 stop being
  acceptable? (Soft limit: 5k visible nodes.)
- **Pane layout**: editor opens (a) overlay center, (b) right of
  sidebar replacing terminal, (c) split with terminal? AFK mode
  (3.4) implications.
