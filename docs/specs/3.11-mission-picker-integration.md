# 3.11 вҖ” Mission Picker Integration with Drafts

## Goal

Reemplazar el modal text-only de "Set mission spec" por un picker que liste los specs publicados (`docs/specs/*.md`) y los drafts (`docs/specs/drafts/*.md`) que ya conoce el motor de Drafts (3.10), conservando el campo de path libre + Browse como escape hatch para specs externos (ej. superpowers).

Hoy el usuario debe escribir o navegar el filesystem aunque la app sabe exactamente quГ© specs existen y dГіnde estГЎn вҖ” fricciГіn innecesaria que el motor de Drafts ya resuelve para sГӯ mismo.

## Out of scope

- Search/filter dentro del picker (con ~10 specs no hace falta v1).
- Permitir drafts como "missions experimentales" вҖ” los drafts siguen sin poder ser misiones; deben publicarse primero (contrato del Operator intacto).
- Cambiar el modo en que el Operator lee el spec (sigue siendo "path вҶ’ archivo en disco").
- Multi-select.
- Persistir un "favorites" o ranking aprendido вҖ” orden es semГЎntico (max ID primero).
- Convertir el panel Drafts en mission picker fullscreen (mantenemos modal para v1, ver decisiГіn en secciГіn Comportamiento).
- SincronizaciГіn con servicios externos (Linear, GitHub) вҖ” especГӯficamente no.

## Acceptance criteria

- [ ] Al disparar Set Mission desde la statusbar / `вҢҳM`, se abre el modal evolutivo con: secciГіn **Published (N)** scrolleable arriba, secciГіn colapsable **Drafts (N)** debajo (default colapsado, oculta si vacГӯa), separador "or pick another file" + text input + Browse abajo, y CTA `[Cancel] [Set mission]`.
- [ ] La secciГіn Published lista todos los specs en `docs/specs/*.md` (excluye `_template.md` y `drafts/`); cada card muestra `ID` mono + `Title` bold + snippet del Goal (вүӨ200 chars). Sort por ID semГЎntica (major/minor) descendente.
- [ ] Click simple sobre una card la marca como `selected`, deshabilita el text input, y habilita "Set mission" usando el path de la card. Doble-click confirma sin pasar por el botГіn.
- [ ] Si la tab activa ya tiene una mission seteada, su card aparece pre-seleccionada con badge `current`.
- [ ] La secciГіn Drafts (cuando expandida) lista los archivos en `docs/specs/drafts/*.md`. Cada item tiene un botГіn **"Publish to use"** (no "Set"); clickearlo cierra el Set Mission modal sin setear, abre el panel Drafts en el wizard de ese slug, y dispara automГЎticamente el modal de Publish del wizard.
- [ ] El text input + Browse mantienen el comportamiento actual; escribir en el input deselecciona cualquier card. "Or pick another fileвҖҰ" como label visible separa esta secciГіn.
- [ ] Atajos de teclado dentro del modal: `вҶ‘/вҶ“` navegan cards publicadas, `Enter` confirma la card seleccionada, `Esc` cancela, `Tab` mueve foco al input para escritura libre.
- [ ] Empty states: sin published specs muestra "No published specs yet. Write one in Drafts (вҢҳвҮ§D)." con link al panel; sin drafts, la secciГіn no aparece.
- [ ] Carga: al abrir el modal, `list_published_specs` y `list_drafts` se disparan en paralelo. Mientras cargan, skeleton de 3 filas con shimmer. Si `list_published_specs` falla, error inline arriba con "Retry"; el text input + Browse siguen funcionando como fallback.
- [ ] Tras publicar un draft via "Publish to use", el toast existente "Open in Set Mission" wirea `mission:set` automГЎticamente вҖ” la tab activa recibe la mission sin re-abrir el Set Mission modal.
- [ ] Tests Rust cubren `parse_published_spec_heading`, `extract_goal_paragraph`, y `list_published_specs_sync` (excluye `_template.md` y `drafts/`, sort correcto). Tests TS cubren la lГіgica `pickerState` (Гәltimo-gana entre card e input) y `canSubmit`.
