# 3.5 — Self-contained docs hub

## Goal

Open an in-app reference panel (⌘?) that explains what each
Covenant feature does — AOM, Agents, Blocks, Recall — without
leaving the terminal. Onboarding without a website.

## Out of scope

- **Editing docs from the UI** — content is static, baked into the
  bundle. Updating a doc means editing the source file and shipping
  a new build.
- **Search** — 4 sections is browseable; search is overkill at this
  size and adds an indexer for nothing.
- **External links navigation** — no embedded browser. Links render
  as plain text or copy-on-click; we do NOT open Tauri windows for
  external URLs.
- **Localization / i18n** — English only for v1. (The rest of
  Covenant is bilingual ad-hoc; formal i18n is a separate effort.)
- **Markdown rendering library** — too heavy for 4 short docs. We
  ship pre-formatted text + light inline formatting (kbd, code
  spans) via small string helpers, NOT marked / markdown-it.
- **Settings page integration** — the docs hub is its own panel,
  parallel to ⌘, not nested in it.
- **Persistence of last-viewed section** — opens to AOM section by
  default each time. Adding "remember last section" is a nice-to-have
  for v2 if anyone asks.

## Acceptance criteria

- [ ] `⌘?` (and `⌘/` as alternate, since `?` requires Shift on most
      keyboards) toggles the docs hub panel.
- [ ] Panel renders an overlay similar to Settings: 220px sidebar
      nav with 4 entries (AOM, Agents, Blocks, Recall) + scrollable
      content area on the right.
- [ ] Clicking a sidebar entry switches the visible content section
      without closing the panel.
- [ ] Each section shows pre-formatted explanatory text covering:
      what the feature is, when to use it, the keyboard shortcut(s),
      and one concrete example.
- [ ] `Esc` closes the panel.
- [ ] Outside-click on the overlay closes the panel.
- [ ] Panel respects the active vibrancy mode (`bg-solid` /
      `bg-vibrant` / `bg-translucent`) — uses `var(--bg-overlay)`,
      not a hardcoded color.
- [ ] `npx tsc --noEmit` passes; no new compiler warnings in
      `cargo check -p covenant`.

## File boundaries

- **Create**:
  - `ui/src/docs/panel.ts` — single class `DocsPanel` with
    open/close/toggle (≤ 250 lines).
  - `ui/src/docs/content/aom.ts` — exports a constant string with
    the AOM section text (≤ 80 lines).
  - `ui/src/docs/content/agents.ts` — same pattern (≤ 80 lines).
  - `ui/src/docs/content/blocks.ts` — same pattern (≤ 80 lines).
  - `ui/src/docs/content/recall.ts` — same pattern (≤ 80 lines).
- **Touch**:
  - `ui/src/main.ts` — wire `⌘?` shortcut + Esc-close handling +
    instantiation. Max 25 lines added, NO existing logic changed.
  - `ui/src/styles.css` — panel styles. Max 180 lines added,
    appended at the end of the file. Reuse existing tokens
    (`--bg-overlay`, `--border`, `--accent`, `--muted`).
- **DO NOT touch**:
  - Anything in `crates/` — this is a frontend-only feature.
  - `ui/src/operator/`, `ui/src/aom/`, `ui/src/recall/`,
    `ui/src/blocks/`, `ui/src/tabs/`, `ui/src/settings/` — the docs
    hub is parallel to all these, not coupled.
  - `ui/src/api.ts` — no new IPC calls; content is bundled.
  - Build configs (`vite.config.ts`, `tauri.conf.json`,
    `Cargo.toml`).

## Complexity

`small` — 1 AOM session, ~3-5 hours of agent work. Static content
+ one panel class + a shortcut wire-up. No backend, no IPC, no
state to persist.

## Open questions

None — the spec is intentionally tight. If the agent encounters
something unclear, that's a signal to ESCALATE rather than guess.
