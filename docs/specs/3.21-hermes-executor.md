# 3.21 тАФ Hermes executor support

## Goal

Recognize and surface [Hermes](https://github.com/NousResearch/hermes-agent)
the same way Covenant already surfaces Claude Code, Copilot CLI, and
Codex: a one-click Spawn preset, an executor chip in the status bar, and
phase detection (Thinking / Running / Writing / Reading / Waiting) so
the operator engine can observe Hermes sessions.

Hermes is a local MIT-licensed CLI (`hermes`) with a multi-provider
backend (OpenRouter, Anthropic, OpenAI, NovitaAI, NVIDIA NIM, Hugging
Face, Nous Portal OAuth). It is **not** a hosted HTTP service тАФ it
launches as a PTY child like every other executor we already support.
That means the integration is purely additive; no architectural changes.

## Out of scope

- **HTTP / remote-agent abstraction.** Hermes has a `hermes gateway` for
  Telegram/Discord/Slack delivery. We're not consuming that тАФ we're
  spawning the interactive CLI. A future remote-agent abstraction is a
  separate spec.
- **OAuth flow for the Nous Portal.** Hermes runs its own `hermes setup`
  wizard for provider auth and stores credentials in its own config.
  We do not re-implement provider management or share Covenant's
  provider keys into Hermes' config. The user runs `hermes setup` once;
  Covenant just launches the binary.
- **Skill / memory inspection UI.** Hermes maintains its own
  agent-curated memory (`/skills`, `/insights`). Surfacing those inside
  Covenant's Project Notes or Operator Mind is a follow-up.
- **Bundled installer.** We don't ship Hermes inside Covenant. The
  preset assumes `hermes` is on `$PATH`; if missing, the spawn fails
  with the existing "command not found" PTY error.

## Acceptance criteria

- [ ] `hermes` (or a path ending in `/hermes`) launched in any tab
      causes the status-bar executor chip to render as `ЁЯдЦ hermes`.
- [ ] A "Hermes" preset appears in the Spawns settings on first run,
      with command `hermes`, no args, no model field (Hermes manages its
      own model via `hermes model`).
- [ ] Phase detection: when Hermes is "Thinking", "Running tool",
      "Writing", or "Reading" the corresponding `ExecutorPhase`
      variant is emitted on the session event bus, observable via the
      existing operator wiring and the activity sidebar.
- [ ] Tests in `crates/blocks/src/executor_phase.rs` cover at least two
      characteristic Hermes output lines per phase variant (captured
      from a real `hermes` run тАФ recorded as fixture strings in the
      test module).
- [ ] Existing executor detection regressions: `claude`, `claude-code`,
      `copilot`, `codex`, `opencode`, `aider`, `cursor`, `pi` still
      detect correctly (no test fails in `ui/src/executor.test.ts`).

## Phase-detection cheat sheet (to be confirmed against real output)

Recommended way to capture: run `hermes` in a Covenant tab, exercise a
few prompts that drive each phase, save the raw PTY transcript via
`covenant trace export <session-id>` (or `script -q out.txt hermes` if
the trace command is missing), and lift the characteristic substrings.

Phases to detect, with first-pass guesses (replace with real strings
once captured):

- **Thinking**: Hermes is a TUI with a spinner; capture the spinner
  marker line, similar to Claude's `HyperspacingтАж` and Copilot's
  `тЧП Thinking`. Likely a single Unicode glyph + label.
- **Running tool**: Hermes streams tool output; look for a tool-call
  banner (e.g. `тЦ╖ bash`, `тЦ╖ web`, `тЦ╖ skill:<name>`) at column 0.
- **Writing / Reading**: Hermes calls into file tools; expect
  `тЦ╖ write_file <path>` / `тЦ╖ read_file <path>` lines or equivalent.
- **Waiting**: in-conversation slash-prompt input. Probably a stable
  prompt-row prefix, e.g. `Hermes тА║ ` or `> `.

Mirror the Copilot matcher structure at `executor_phase.rs:108-188`
exactly тАФ same `once_cell` regex pattern, same `apply_line` integration.

## Open questions

These should not be silently decided:

- **Hermes is missing from $PATH on a fresh machine тАФ what do we do?**
  Two options: (a) leave the preset; spawn fails visibly with the
  shell's "command not found"; ship a tooltip on the preset row that
  links to `hermes-agent.nousresearch.com/docs/getting-started/installation`;
  (b) detect absence at app start (`which hermes`) and grey out the
  preset with an install CTA. Pick one before merge.
- **Should the Hermes spawn preset be `default: false` (current
  Copilot/Codex behavior) or promoted via a release-note callout?**
  Leaning false тАФ Claude remains the default.
- **Branded icon vs. generic.** Nous Research has a wordmark/logo; do
  we ship an SVG with attribution, or fall back to the generic
  `Icons.zap` / `Icons.robot`? If branded, confirm the license
  (presumably MIT-aligned given the agent itself is MIT).
- **Phase regex stability across Hermes versions.** Hermes ships
  rapidly. Decide whether the regex matchers gate on a TUI version
  header (if Hermes prints one) or simply best-effort match тАФ same
  policy we use for Claude Code's spinner.
