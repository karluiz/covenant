Why this works

  • Information-dense. Each row shows name, endpoint, status, what it's used for, and last test result. The current design hides most of this behind cards.
  • Sortable / filterable. Click a column header to sort by status (failing first), kind, last-test. Mental model: this is your provider fleet, not a list of forms.
  • Inline expand to edit. No modal, no drawer — the row opens in place. Two-column layout inside: form on the left, health diagnostics on the right (last test response, catalog size, etc.). Diagnostics are first-class, not buried.
  • Keyboard-first. Arrow keys + ↵ expand + T to test + ⌫ delete. Power users never reach for the mouse.
  • "Used for" column. The crosswalk between providers and what they actually serve (Triage / Reasoning / Tools / Operators / Local) is visible at the list level — fixes the disconnect with the current Models routing tab.
  • Tradeoff: least friendly to first-time users — a table reads as "advanced settings". Best after 3+ providers are configured; with one, it's overkill compared to Direction A or B.
