Why this works

  • Scales beyond a wall of cards. Five providers fit in the rail at a glance; fifty still fit with a scroll. Today's stack forces vertical hunt.
  • State legible at a glance. Status dot per row (green / yellow / red) tells you what's working without clicking anything. Endpoint preview disambiguates duplicates (two openai-compat providers? endpoint shows which is which).
  • Provider identity. Colored glyph anchors each kind — Anthropic orange, Azure blue, LM Studio violet. The kind tag (azure_foundry) moves out of the title into a muted subtitle.
  • Secrets handled properly. Password input with explicit Reveal + Copy buttons; no more 88-char wall of bullets.
  • Forward-looking section: Use as default for. The same place users land to add a provider also lets them route Triage / Reasoning / Tools / Operators — short-circuits a separate "models routing" detour.
  • Tradeoff: two-pane layout needs ≥ ~640 px width to read comfortably. Narrow window → fall back to Direction B style stack.
