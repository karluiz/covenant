# AGENTS.md — Covenant

AI-native terminal: Tauri 2 + Rust + xterm.js. Shell runs inside a PTY owned by the Rust backend.
Primary platform: macOS. Linux (Ubuntu 24.04+, Fedora 39+, Bazzite) is supported. See `CLAUDE.md` for full architecture, milestones, and hard constraints.

---

## Commands

### Dev
```bash
npm install           # must run before anything else
npm run tauri:dev     # vite on :1420 + Rust HMR (starts both automatically)
```

### Build
```bash
npm run tauri:build          # full .app + .dmg (macOS) or .deb/.rpm/AppImage (Linux)
npx tauri build --bundles app  # .app only, faster (macOS)
```

### Install to /Applications (macOS)
```bash
npm run install:app          # build + quit running instance + install
npm run install:app:reuse    # skip build, install existing bundle
npm run install:app:open     # build + install + launch
```
The macOS install script quits Covenant via AppleScript, strips quarantine xattr, and keeps 5 backups under `~/Library/Application Support/com.karluiz.covenant/backups/`.

### Install on Linux
```bash
./scripts/install-linux.sh           # auto-detect format (AppImage/deb/rpm), backup, install
./scripts/install-linux.sh --format appimage --open   # install AppImage and launch
```
Data backup goes to `$XDG_CONFIG_HOME/com.karluiz.covenant/backups/`.

### Frontend only (no Rust)
```bash
npm run dev      # vite dev server only
npm run build    # tsc + vite → dist/
```

## Linux build prerequisites

Install before running `npm run tauri:build` on Linux:

```bash
# Ubuntu 24.04+ / Debian
sudo apt install \
    libwebkit2gtk-4.1-dev libssl-dev libgtk-3-dev \
    libayatana-appindicator3-dev librsvg2-dev build-essential

# Fedora 39+
sudo dnf install \
    webkit2gtk4.1-devel openssl-devel gtk3-devel \
    libappindicator-gtk3-devel librsvg2-devel
```

**Ubuntu 22.04 is NOT supported** — it ships `webkit2gtk` 2.38 (Tauri 2 needs ≥ 2.40).

## Architecture Gotchas

### Crate structure
```
covenant (app binary)
  ├── karl-pty        PTY abstraction over portable-pty
  ├── karl-blocks     OSC 133 parser (uses vte, NOT a hand-rolled VT parser)
  ├── karl-session    Session lifecycle + tokio::broadcast event bus
  ├── karl-agent      Anthropic HTTP streaming client
  └── karl-familiar   Per-session AI companion with SQLite memory
```

The app crate lib name is `covenant_lib` (not `covenant`) to avoid a macOS linker conflict.
`crate-type = ["staticlib", "cdylib", "rlib"]` — required by Tauri on macOS.

### Tauri config — two files
```
crates/app/tauri.conf.json         ← base config (macOS defaults: transparent, hudWindow)
crates/app/tauri.linux.conf.json   ← Linux override: transparent:false, no windowEffects
crates/app/capabilities/default.json
crates/app/gen/                    ← gitignored; auto-generated by Tauri CLI
```
`frontendDist` points to `../../dist` (workspace root `dist/`). Vite `root` is `ui/`; `outDir` is `../dist`.
Tauri deep-merges `tauri.linux.conf.json` on Linux builds (RFC 7396). Arrays replace, scalars merge.

### Shell detection — multi-shell support
`$SHELL` env var controls which shell spawns. Three injection mechanisms:
- **zsh**: private `ZDOTDIR` with shim rc files + `--no-globalrcs`
- **bash**: `--rcfile <tmp>/bashrc` shim that chains `~/.bashrc`
- **fish**: synthetic `XDG_CONFIG_HOME` pointing to a shim `fish/config.fish`
- **other**: shell spawns unaugmented (no OSC 133 blocks — terminal works, AI layer blind)

All three shell snippets are bundled via `include_str!` in `crates/app/src/lib.rs`.
**Never hard-code `/bin/zsh`** — it isn't the default on Ubuntu or Fedora.

### Globals injected by Vite at build time
- `__APP_VERSION__` — from `package.json`
- `__APP_CHANGELOG__` — full contents of `CHANGELOG.md` inlined as a string (sidesteps `file://` read issues on macOS)

Declared in `ui/src/globals.d.ts`.

### TypeScript IPC boundary
All Tauri `invoke` calls must go through `ui/src/api.ts` with typed return values. Direct `invoke` calls elsewhere are an architectural violation.

### Window background — platform defaults
- **macOS**: `WindowBackground::Vibrant` (default) — NSVisualEffectView `hudWindow` effect
- **Linux**: `WindowBackground::Solid` (default) — no system blur API; transparent window causes black artefacts without a compositor. The CSS `body.bg-solid` rule sets an explicit `background: #0b0d10`.

### Shell integration
`shell-integration/osc133.zsh`, `osc133.bash`, and `osc133.fish` are all bundled via `include_str!` in `crates/app/src/lib.rs`. On session spawn, the detected shell selects the appropriate dotdir strategy — user dotfiles are **never modified**. The zsh snippet auto-loads `zsh-autosuggestions` from common install locations.

### Familiar observer thread
`rusqlite::Connection` is `!Sync` and the future is `!Send`, so the familiar observer runs on a dedicated OS thread with its own `current_thread` Tokio runtime — not on the main async executor.

### SQLite / embeddings
- History DB: `app_config_dir()/history.db` (WAL, foreign keys ON) — macOS: `~/Library/Application Support/com.karluiz.covenant/`; Linux: `~/.config/com.karluiz.covenant/`
- `sqlite-vec` extension must be registered via `sqlite3_auto_extension` before any `Connection::open`
- Familiar DBs: `~/.karlTerminal/familiars/<id>.sqlite`
- `fastembed::TextEmbedding::embed` requires `&mut self` → wrap in `Mutex<TextEmbedding>` inside `Arc`
- Both `sqlite-vec` and `fastembed` (ONNX Runtime) have Linux x86_64/aarch64 pre-built binaries — no extra setup needed.

## Rust Conventions

- `thiserror` in library crates; `anyhow` only in `crates/app` (the binary boundary).
- No `unwrap()` outside `#[cfg(test)]` and `main()`.
- PTY reader: synchronous, runs in `tokio::task::spawn_blocking`; pushes bytes into `mpsc::channel`; async side drains and broadcasts. Never block the reader.
- IDs: `ulid::Ulid` newtyped — `SessionId(Ulid)`, `BlockId(Ulid)`.
- Logging: `tracing` with `session_id`/`block_id` as structured fields, never string-interpolated.
- Public types: `Debug + Clone`; events: `Serialize`.

## Safety / Security Rules

### `crates/app/src/safety.rs`
Hard blocklist for autonomous execution. **Adding is fine; removing requires written justification in the code review.**

### Secret masking
Call `safety::mask_secrets(text)` before any text reaches the LLM or is persisted. Patterns masked: `sk-ant-*`, `sk-*`, `ghp_/ghu_/gpo_/gpr_/ghs_`, AWS AKIA/ASIA, JWT, PEM keys, etc.

### Agent dispatch
`agent::dispatch()` is the **only** path to the Anthropic API. Call it; never call the HTTP client directly.

### Execution policies (default: `SuggestOnly`)
`SuggestOnly` → `Allowlist(regex_set)` → `ConfirmEach` → `FullAuto { sandbox }`

Rate limiter: 6 calls/minute per session (sliding 60 s window), enforced in `AppState.rate`.

## Do Not

1. Reimplement VT/ANSI parsing. Use `vte`/`vt100`; let xterm.js render.
2. Poll stdout. Use the `spawn_blocking` reader task + `mpsc`.
3. Parse blocks without OSC 133 markers — heuristics always break.
4. Block the PTY reader (no `.await`, no sync I/O inside it).
5. Send raw ANSI to the LLM — strip first.
6. Call the Anthropic API outside `agent::dispatch()`.
7. Store or log unmasked secrets.
8. Add React, Electron, Webpack, or other heavyweight frontend deps.
9. Re-mount the xterm.js instance on prop change — use refs, single mount.
10. Use `npm` alternatives (pnpm, yarn) — `package-lock.json` is committed.
11. Hard-code `/bin/zsh` — use `detect_user_shell()` or `SpawnOptions::for_shell(&shell)`.
