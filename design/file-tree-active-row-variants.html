File tree — active-row treatments

Three visual treatments for the currently-open file in the structure tree. The "active" row in each mockup is the same file: docs/superpowers/plans/2026-05-04-aom-exclusion-visibility.md. All three treatments would be combined with auto-expand of collapsed parents + scroll-into-view on editor.open().

A · Background tint + accent stripe (recommended)

Subtle accent-tinted background + 2px left stripe. Visible while scrolling. Pattern used by VSCode, Zed, Sublime.

Behavior layered on top of all three variants:

  • On editor.open(path) → mark row active, auto-expand collapsed ancestors so the row is reachable, scrollIntoView({ block: "nearest" }).
  • On editor.close() → clear the active row.
  • Reveal fires once per open. User can collapse / scroll freely after that without the tree fighting them.
  • Per-tab — each tab has its own StructureTree + editor pairing, so the highlight follows the tab.

Accent color is var(--accent, #b794f4) — already defined and used elsewhere in the app (matches the structure-context-menu hover tint at styles.css:8034).
