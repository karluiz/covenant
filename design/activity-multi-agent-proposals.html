Variant A — chip strip · multi-select

Best for2–5 agents you want to glance at the same time. Quick toggles in/out. Default state"All" pill is hot; the per-agent chips are dim. Click one → solo it (ring lights up in agent color). Cmd-click → add to selection. WinsStatus (busy/idle) is always visible inline on the chip. Lowest click-cost to compare two agents. RisksEats ~36px vertical; wraps to 2 rows past ~5 agents and starts to look messy.

Variant B — tab bar · single-select

Best forYou usually want to focus on one agent at a time. Combined view is rare. Default state"All" tab selected; switch by clicking any tab. The header rephrases ("Claude · ACTIVE · ▸ writing patch") so the panel feels like a per-agent inspector. WinsFamiliar (matches the existing Chat/Tasks/Activity tab strip). Per-agent counts are obvious. Cheap horizontal scroll for >5 agents. RisksNo multi-select — can't isolate "Claude + Codex" without going back to All.

Variant C — collapsed picker · dropdown

Best forLots of agents (10+), or when vertical space in the panel is tight. Default stateStacked avatars + selection label ("Claude + Codex"). Click → multi-select dropdown with counts and an "All agents" top option. WinsScales to N agents without overflow. Mirrors GitHub's reviewers/assignees picker — instantly familiar. RisksSelection isn't visible until you open it. Two clicks to swap from solo Claude → solo Mibli (open, uncheck, check, close).

Recommendation

Start with A (chip strip) because today you have ≤5 agents and the always-visible status dots are valuable signal — you'd see "Claude is busy" without clicking anything. Migrate to C later if the agent count grows past comfortable chip-strip width. B is a fallback if multi-select feels like overkill and you really do focus on one at a time.
