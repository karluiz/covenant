Right-click → run a saved command / prompt

Three ways to surface saved commands (per-group) and prompts (global) inside the terminal context menu, below the existing Split actions. All use the app's real menu styling. The blue row is the hover state.

A · Nested flyout
Two parent items expand a side flyout on hover. Keeps the root menu tiny; scales to any number of saved items. Requires extending contextMenu.ts with submenu support.
B · Quick picker
One unified, type-to-filter list of commands + prompts (tagged by kind). Fastest when you have many — no hunting through nested menus. Slightly heavier menu; needs a tiny filter input.
C · Flat sections
Commands
Prompts
Everything inline under labeled sections — zero extra interaction, one glance. Simplest to build (no submenu/search). Gets tall if you have many saved items, so best capped to a top-N.
